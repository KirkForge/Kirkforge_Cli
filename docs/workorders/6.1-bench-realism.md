# Workorder 6.1 — Fix Benchmark Harness Realism

## Status: Planned

## Problem

The bench executor (`src/session/bench.rs:141`) constructs a
`CompositeToolset::empty()` for every task. This means the model has no
file-edit or bash tools available during benchmark runs. Five of the 10 tasks
target the real repo (`add_adr`, `add_cli_flag`, `add_test_for_function`,
`fix_clippy_warning`, `refactor_extract_function`) and are effectively unwinnable
as written — the model cannot edit files without tools. The bench results
therefore measure toolset-presence, not agent capability.

Additionally, the `add_adr` task's verify spec checks for
`docs/adr/039-task-benchmark-harness.md` but the actual benchmark ADR is
`038-task-benchmark-harness.md`. The task is stale and would fail verification
regardless of model capability.

## Root cause

The empty toolset was a minimal-safety choice during the bench harness build
(P0). It was never revisited when real-repo tasks were added.

## Files to touch

- `src/session/bench.rs` — replace `CompositeToolset::empty()` with a sandboxed
  toolset that includes `read_file`, `write_file`, `edit_file`, `bash`, `glob`,
  `grep`. All operations constrained to the temp sandbox dir.
- `benches/tasks/add_adr.toml` — fix the verify path from `039-` to `038-`, or
  change the task to create a new ADR with a non-conflicting number.
- `crates/kirkforge-bench/src/lib.rs` — no change needed (verify logic is
  correct; the task file is wrong).
- `docs/adr/038-task-benchmark-harness.md` — update the "Implementation notes"
  section to reflect that the executor now provides a real toolset.

## Approach

1. In `bench.rs`, build a `CompositeToolset` that registers the 6 sandboxed
   tools with the sandbox dir set to the tempdir. The auto-approval task already
   approves everything, so no approval-gate changes needed.
2. Keep the timeout and per-task sandbox isolation.
3. Fix the `add_adr` task to verify a file that makes sense (either fix the
   number or change the task to write a fresh ADR with a higher number and
   verify it contains `Status: Accepted`).
4. Run `kirkforge bench --tasks benches/tasks/ --model qwen2.5:0.5b` locally to
   confirm tasks are now winnable (at least the easy ones).

## Gate

```
cargo fmt --check && cargo check --workspace --all-targets &&
cargo clippy --all-targets -- -D warnings &&
cargo test --locked --workspace --no-fail-fast
```

Plus: `kirkforge bench --tasks benches/tasks/ --model qwen2.5:0.5b --summary /tmp/bench-smoke.md`
and confirm at least 3/10 tasks pass (the easy ones should be winnable now).

## Done condition

- Bench executor provides real sandboxed tools.
- `add_adr` task verify path is correct.
- Full gate green.
- Local bench run shows improved pass rate vs the empty-toolset baseline.
