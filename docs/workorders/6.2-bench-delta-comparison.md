# Workorder 6.2 — Bench Baseline Storage and Delta Comparison

## Status: Planned

## Problem

The bench harness produces a single-run `BenchReport` (JSON + markdown) but has
no way to compare two reports. There is no baseline storage, no delta
computation, and no `kirkforge bench compare` subcommand. ADR-038 claims "every
PR that touches agent code gets a benchmark delta" but the delta infrastructure
does not exist.

## Root cause

The bench crate was built with single-run reporting as the MVP. Comparison was
always the intent (ADR-038) but was deferred.

## Files to touch

- `crates/kirkforge-bench/src/lib.rs` — add:
  - `compare_reports(baseline: &BenchReport, current: &BenchReport) -> DeltaReport`
  - `DeltaReport` struct: per-task `TaskDelta { name, difficulty, baseline_success, current_success, delta_tokens_in, delta_tokens_out, delta_duration_secs, delta_cost_usd }`
  - `write_markdown_delta(delta: &DeltaReport, path: &Path)` — markdown table with delta columns
  - Unit tests for `compare_reports` (regression, improvement, no-change cases)
- `src/cli.rs` — add `bench compare --baseline <json> --current <json> --summary <md>` subcommand
- `src/main/mod.rs` — add `handle_bench_compare_command`
- `benches/baselines/` (new, gitignored) — local storage for baseline reports; the CI workflow will store the canonical baseline
- `.gitignore` — add `benches/baselines/`

## Approach

1. `compare_reports` matches tasks by name. For each task present in both
   reports, compute deltas. Tasks only in one report are flagged as
   added/removed.
2. `DeltaReport` includes an aggregate summary: success rate change, total token
   delta, total cost delta.
3. The markdown delta writer produces a table:

   ```
   | Task       | Diff | Baseline | PR  | Δtokens | Δcost   |
   |------------|------|----------|-----|---------|---------|
   | add_adr    | hard | fail     | pass| -1.2k   | -$0.001 |
   ```

4. The `bench compare` CLI subcommand loads two JSON reports, calls
   `compare_reports`, writes the delta markdown.
