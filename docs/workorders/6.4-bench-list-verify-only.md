# Workorder 6.4 — Bench List and Verify-Only Subcommands

## Status: Planned

## Problem

The `kirkforge bench` CLI has only one mode: run all tasks with a live LLM. There
is no way to list tasks or verify task definitions without spending model tokens.
A `verify-only` mode would let CI smoke-test task validity (do the setup files
parse, do the verify commands work) in seconds rather than minutes.

## Root cause

The CLI was built with the single run-all mode as MVP.

## Files to touch

- `src/cli.rs` — add subcommands:
  - `kirkforge bench list [--tasks <dir>]` — list all task names, difficulties, and verify types
  - `kirkforge bench verify-only [--tasks <dir>] [--task <name>]` — run only the verify step (skip the LLM run); for each task, write the setup files to a temp dir and run the verify command, reporting pass/fail
- `src/main/mod.rs` — add `handle_bench_list_command` and `handle_bench_verify_only_command`
- `crates/kirkforge-bench/src/lib.rs` — add:
  - `fn list_tasks(dir: &Path) -> Vec<TaskInfo>` where `TaskInfo { name, difficulty, verify_type }`
  - `fn verify_only(task: &BenchTask, sandbox: &Path) -> TaskResult` (already exists as `verify_task` — just needs a CLI wrapper)
- Unit tests for `list_tasks`.

## Approach

1. `list_tasks` loads all TOML files and extracts metadata without running
   anything.
2. `verify-only` writes the setup files to a temp dir, runs the verify command,
   and reports the result. This is deterministic and fast (no LLM).
3. For tasks with no `setup` (real-repo tasks), `verify-only` skips them with a
   "needs real repo" note — or optionally runs the verify command against the
   current repo (useful for checking if a task's verify command is still valid).
