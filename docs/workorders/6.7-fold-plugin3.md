# Workorder 6.7 — Fold Plugin3 into Core

## Status: Planned

## Problem

Plugin3 (`crates/plugin3-core`) is the token-budget guard. It is not linked into
the main binary and is invoked via shell scripts. Worse, Plugin3's 4 hooks
emit canned JSON under KirkForge because the host passes only env vars, not full
event context. The budget logic cannot see real `post-tool-bash` results or
`pre-compact` history. Folding it in eliminates this lossy shim and gives the
budget system full access to session state.

## Root cause

Same as Stratum — standalone CLI plugin added to the workspace but never linked
as a library dependency.

## Files to touch

- `Cargo.toml` (root) — add `plugin3-core = { path = "crates/plugin3-core", optional = true }` and feature `budget = ["dep:plugin3-core"]`; add `budget` to `default`.
- `src/shared/config/tools.rs` — add budget config fields (`budget_ceiling`, `budget_approaching_ratio`) or wire them through the existing `SessionConfig`.
- `src/session/plugin_tools/loader.rs` — when the `budget` feature is enabled, register Plugin3's 7 tools as direct Rust calls.
- `src/session/hooks.rs` — when the `budget` feature is enabled, register the 4 hooks as in-process Rust handlers **with full event context** (this is the key win — the hooks can now see real tool results and pre-compact history).
- `src/session/budget.rs` (new) — a module wrapping `plugin3_core::budget`, `plugin3_core::orchestrator`, `plugin3_core::slicing`, `plugin3_core::store`. Exposes 7 functions matching the tool names.
- `src/session/executor/turn.rs` — the `post-tool-bash` and `post-tool-write_file` hooks now receive the actual tool result content (not just env vars). Wire the budget check into the turn loop so oversized outputs are sliced *before* being appended to the conversation.
- `src/session/prompt/microcompaction.rs` — the `pre-compact` hook now receives the actual conversation history. Wire the budget system's compaction trigger.
- `src/session/plugin_tools/tests.rs` — update tests.
- `docs/adr/0047-plugin3-fold-in.md` (new) — pin the decision.

## Approach

1. Add `plugin3-core` as an optional dep + feature.
2. Create `src/session/budget.rs` with 7 functions calling `plugin3_core` directly.
3. The critical change: the 4 hooks become in-process handlers that receive
   `&BusEvent` (or the relevant event data) instead of just env vars. This
   means:
   - `post-tool-bash`: receives the bash output, can check if it's oversized
     and slice it before it enters the conversation.
   - `post-tool-write_file`: receives the file content, same check.
   - `pre-compact`: receives the conversation history, can make informed
     compaction decisions.
   - `session-start`: initializes the budget state.
4. Wire the budget check into `turn.rs` after tool results are collected but
   before they're appended to the conversation. If the budget is exceeded,
   slice/compact the result first.
5. The `plugin3` binary remains as a standalone CLI.

## Done condition

### Shipped
- `plugin3-core` is linked when `--features budget`.
- The 7 tools are direct Rust calls.
- The 4 hooks are in-process Rust handlers with **full event context** (the headline win — the lossy canned-JSON shim is eliminated; hooks now get real event context instead of env vars):
  - `SessionStartHook` (session-start) — logs budget state at session start.
  - `PostToolBashHook` (post-tool-bash) — receives the real tool result content via `HookContext.tool_result`, estimates tokens (len/4), records to `TokenBudget`, and warns if approaching/over.
  - `PostToolWriteFileHook` (post-tool-write_file) — same as post-tool-bash but for `write_file`.
  - `PreCompactHook` (pre-compact) — receives compact stats via `HookContext.compact_stats`, resets `budget.used` to 0 if the budget was over/approaching.
  - All 4 share a process-global `TokenBudget` via `OnceLock` with the budget tools.
  - Registered under `#[cfg(feature = "budget")]` in `src/session/executor/mod.rs`. The executor's `run_hook_with_result` and `run_compact_hook` now pass the full `HookContext`.
- ADR-047 pins the decision.
- Drift tests pass.
- Full gate green with and without the feature.
- Binary size impact measured (expected: small — `plugin3-core` adds `blake3`, `chrono`, both already transitive deps).

### Deferred (honestly — see ADR-047)
- The `budget_ceiling` and `budget_approaching_ratio` config fields are deferred (not needed for MVP — the defaults of 200K and 0.8 are used).
- The hooks record usage and warn, but do not yet slice/compact tool results before they enter the conversation (the budget check observes and reports; it does not mutate the turn output).
