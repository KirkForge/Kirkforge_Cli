# Workorder 6.8 — Fold Draw into Core

## Status: Planned

## Problem

Draw (`crates/kirkforge-draw-core`) is a pure document model for terminal
diagrams. It is not linked into the main binary. The `draw_render` tool shells
out to the `kfd` binary. The `post-turn` hook is pure bash (no binary) but still
goes through the shell-hook path.

## Root cause

Same pattern as Stratum and Plugin3 — standalone plugin never linked as a
library dependency.

## Files to touch

- `Cargo.toml` (root) — add `kirkforge-draw-core = { path = "crates/kirkforge-draw-core", optional = true }` and feature `draw = ["dep:kirkforge-draw-core"]`; add `draw` to `default`.
- `src/session/plugin_tools/loader.rs` — when the `draw` feature is enabled, register `draw_render` as a direct Rust call.
- `src/session/hooks.rs` — when the `draw` feature is enabled, register the `post-turn` hook as an in-process Rust handler (scan for new `.td.json` files).
- `src/session/draw.rs` (new) — a module wrapping `kirkforge_draw_core::render_text` and `kirkforge_draw_core::doc`. Exposes a `render(path: &str) -> String` function that loads a `.td.json` and returns fenced markdown.
- `src/session/plugin_tools/tests.rs` — update tests.
- `docs/adr/0048-draw-fold-in.md` (new) — pin the decision.

## Approach

1. Add `kirkforge-draw-core` as an optional dep + feature. Note: we only need
   `-core` (the pure document model), not `kirkforge-draw` (the TUI binary).
   `-core` adds `unicode-segmentation` and `unicode-width` — both lightweight.
2. Create `src/session/draw.rs` with a `render` function that calls
   `kirkforge_draw_core::render_text::build_scene` + `render_plain` and wraps
   the output in a fenced code block.
3. Register `draw_render` as a `DrawToolWrapper` implementing `Tool`.
4. Register the `post-turn` hook as a Rust closure that globs for `.td.json`
   files and emits a suggestion if any are new.
5. The `kfd` binary remains as a standalone interactive editor.

## Gate

```
cargo fmt --check && cargo check --workspace --all-targets --features draw &&
cargo clippy --all-targets --features draw -- -D warnings &&
cargo test --locked --workspace --no-fail-fast --features draw
```

Also without the feature:
```
cargo check --workspace --all-targets
cargo test --locked --workspace --no-fail-fast
```

## Done condition

- `kirkforge-draw-core` is linked when `--features draw`.
- `draw_render` is a direct Rust call (no subprocess).
- The `post-turn` hook is in-process.
- ADR-048 pins the decision.
- Drift tests pass.
- Full gate green with and without the feature.
- Binary size impact measured (expected: small — `unicode-segmentation` +
  `unicode-width` are lightweight).
- Depends on: Workorder 6.6 (fold-in pattern established).
