# Workorder 6.9 — Fold Video into Core (Non-Default Feature)

## Status: Planned

## Problem

Video (`crates/kirkforge-video`) is the heaviest satellite crate. It pulls
`serde_yaml`, `strum`, `which`, and `tracing-subscriber` — dependencies that are
new to the root dep set. Folding it into core behind a **non-default** feature
flag lets users who want agent-driven video editing opt in without bloating the
default binary.

## Root cause

Same pattern as the other satellites, but with a heavier dep footprint that
justifies a non-default feature.

## Files to touch

- `Cargo.toml` (root) — add `kirkforge-video = { path = "crates/kirkforge-video", optional = true }` and feature `video = ["dep:kirkforge-video"]`. **Do NOT add to `default`** — video is opt-in.
- `src/shared/config/tools.rs` — the `kirkforge-video` entry in `default_plugin_sources` should be gated: when the `video` feature is off, it should not be in the default list.
- `src/session/plugin_tools/loader.rs` — when the `video` feature is enabled, register the 8 video tools as direct Rust calls. When disabled, skip the video plugin entirely (no shell fallback — the binary won't have the code).
- `src/session/video.rs` (new) — a module wrapping `kirkforge_video`'s public API. Exposes 8 functions matching the tool names.
- `src/session/plugin_tools/tests.rs` — update tests: the `default_plugin_sources_are_present_and_loadable` test must handle the video plugin being absent when the feature is off.
- `docs/adr/0049-video-fold-in.md` (new) — pin the decision, including the binary-size justification for the non-default feature and the new transitive deps.

## Approach

1. Add `kirkforge-video` as an optional dep + non-default feature.
2. Create `src/session/video.rs` with 8 functions calling `kirkforge_video`'s
   API directly.
3. Register the 8 tools as `VideoToolWrapper` implementing `Tool`.
4. Video has no hooks — only tools.
5. When the feature is off, the video plugin is not registered at all. The
   `kirkforge-video` binary remains as a standalone CLI for non-KirkForge use.
6. Document the new deps in the ADR: `serde_yaml`, `strum`, `which` are new to
   the root. Justify: video editing requires YAML config parsing (`serde_yaml`),
   enum derive macros (`strum`), and FFmpeg binary discovery (`which`). These
   are only compiled when the user opts in via `--features video`.

## Gate

```
cargo fmt --check && cargo check --workspace --all-targets --features video &&
cargo clippy --all-targets --features video -- -D warnings &&
cargo test --locked --workspace --no-fail-fast --features video
```

Also without the feature (the default build):
```
cargo check --workspace --all-targets
cargo clippy --all-targets -- -D warnings
cargo test --locked --workspace --no-fail-fast
```

## Done condition

- `kirkforge-video` is linked when `--features video`.
- The 8 video tools are direct Rust calls.
- The default build (no `--features video`) does not include video and its deps.
- ADR-049 pins the decision with dep-size justification.
- Drift tests pass.
- Full gate green with and without the feature.
- Binary size impact measured and documented: default build unchanged; `--features video` build size increase recorded.
- Depends on: Workorder 6.6 (fold-in pattern established).
